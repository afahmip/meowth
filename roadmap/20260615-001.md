# Feature: [Transactions with multiple items]

## Requirements
- One transaction might have multiple items. For example: 1 grocery run receipt might have items with individual prices. Or 1 bank transfer has no other items.
- Each items or transactions can have their own category.
- Transactions can be an income or expense.

## User Flow / Behavior
- Users can fetch transactions with optional filters by category ID, date range, or keywords.
- Users can add a new transaction or new items to existing transaction.
- Users can also edit existing transaction or items inside transaction.
- Users can fetch list of categories.
- Users can add / edit a category.
