# Feature: [Transactions accounts]

## Requirements
- Account is an entity that stores balance and owns transactions. For example: transaction X is transfered from bank account A.
- Accounts can be a bank, e-wallet (e.g. Apple Pay, etc), or cash.
- Each transactions must be owned by an account (mandatory).
- Each transactions might be transfered to other account or not. For example: (1) money movement from my bank A to my wallet B, (2) groceries purchase paid by bank A to the grocery store (not an account).

## User Flow / Behavior
- Users can create or edit accounts.
- Users can create or edit transactions (now with account ID).
- Users can fetch list of accounts.
- Users can fetch list of transactions (now with account info).
